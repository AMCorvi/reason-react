Children

Les composants enfants JSX sont de type array. Cela pose un problème, parce que le code <Foo> children </Foo> pour lequel children provient des propriétés (càd votre fonction make) est vraiment maintenant un tableau (Foo.make [| children |]) et a un type incompatible.

Certaines personnes contournent cela en écrivant <Foo> (ReasonReact.arrayToElement children) </Foo>, mais c’est incorrect ; des alertes de clés ReactJS vont être générées. Instead, you should use the desugared form, Foo.make children, which doesn't automatically wrap the children in an array wrapper like JSX does. Learn about desugaring JSX here.

Pour le passer en tant qu'élément DOM, ex. <div foo=bar> children </div> utilisez la syntaxe suivante :

ReasonReact.createDomElement "div" props::{"foo": bar} children

La raison qui fait que passed des tableaux imbriqués d'enfants en ReactJS ne produit pas d'alerte est que ReactJS utilise quelques combines en interne.

Ce n’est pas idéal ; nous allons améliorer cela à l’avenir !
