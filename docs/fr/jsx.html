JSX

Reason embarque la syntaxe JSX! ReasonReact effectue la transformation depuis un appel de fonction agnostique en un appel spécifique ReasonReact via une macro. Pour tirer parti du JSX ReasonReact, mettre {"reason": {"react-jsx": 2} dans votre bsconfig.json (schéma ici).

À Noter qu’en raison des contraintes actuelles concernant la syntaxe, il est nécessaire de mettre des espaces autour des enfants JSX: <div> foo </div>.

Minuscules #

<div foo={bar}> {child1} {child2} </div>

est transformé en

ReactDOMRe.createElement "div" props::(ReactDOMRe.props foo::bar ()) [|child1, child2|]

qui est compilé en code JS:

React.createElement('div', {foo: bar}, child1, child2)

<div /> sans propriétés est transformé en:

ReactDOMRe.createElement "div" [|child1, child2|]

qui est compilé en

React.createElement('div', undefined, child1, child2)

Majuscules #

<MyReasonComponent key={a} ref={b} foo={bar} baz={qux}> {child1} {child2} </MyReasonComponent>

est transformé en

ReasonReact.element key::a ref::b (MyReasonComponent.make foo::bar baz::qux [|child1, child2|])

<MyReasonComponent /> sans propriétés est transformé en:

ReasonReact.element (MyReasonComponent.make [||])

Le make ci-dessus est exactement la fonction make vue dans les sections précédentes.

Notez comment ref et key ont été transformées depuis l’appel JSX vers l’appel ReasonReact.element. ref et key sont des mots réservés en ReasonReact, comme dans ReactJS. Ne pas les utiliser comme propriétés dans votre composant !
