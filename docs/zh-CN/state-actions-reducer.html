State, Actions & Reducer

终于，我们要讨论有状态的组件了！

ReasonReact有状态组件和ReactJS中的有状态组件类似，只是多了原生的"reducer"的概念(like Redux)。 如果你不了解Reducer，暂且将它理解为一个状态管理系统。 如果还是不太明白的化，暂且先认为：这东西真牛！

To declare a stateful ReasonReact component, instead of ReasonReact.statelessComponent("MyComponentName"), use ReasonReact.reducerComponent("MyComponentName").

let component = ReasonReact.reducerComponent("Greeting");

let make = (~name, _children) => {
  ...component,
  initialState: () => 0, /* here, state is an `int` */
  render: (self) => {
    let greeting =
      "Hello " ++ name ++ ". You've clicked the button " ++ string_of_int(self.state) ++ " time(s)!";
    <div> {ReasonReact.stringToElement(greeting)} </div>
  }
};

initialState

ReactJS中的 getInitialState 在ReasonReact里被称为 initialState。 它的输入参数类型是unit，返回类型是state type， 这个state的type可以是任何东西！ 一个Int型，一个普通Record类型的ref等等，但总之你应该在写 reducerComponent 之前定义好：

type state = {counter: int, showPopUp: bool};

let component = ReasonReact.reducerComponent("Dialog");

let make = (~onClick, _children) => {
  ...component,
  initialState: () => {counter: 0, showPopUp: false},
  render: ...
};

由于props只是 make 函数的参数，你可以很方便的用它来构造初始的state。

Actions & Reducer

在 ReactJS 里，我们会像这样来更新state：

{
  ...
  handleClick: function() {
    this.setState({count: this.state.count + 1});
  },
  handleSubmit: function() {
    this.setState(...);
  },
  render: function() {
    return (
      <MyForm
        onClick={this.handleClick}
        onSubmit={this.handleSubmit} />
    );
  }
}

在 ReasonReact 里，我们会把所有处理state的逻辑集中到一起，这就是组件的 reducer ！先来看一个例子，稍后我们会具体解释：

type action =
  | Click
  | Toggle;

type state = {
  count: int,
  show: bool
};

let component = ReasonReact.reducerComponent("MyForm");

let make = (_children) => {
  ...component,
  initialState: () => {count: 0, show: false},
  reducer: (action, state) =>
    switch action {
    | Click => ReasonReact.Update({...state, count: state.count + 1})
    | Toggle => ReasonReact.Update({...state, show: ! state.show})
    },
  render: (self) => {
    let message = "Clicked " ++ string_of_int(self.state.count) ++ " times(s)";
    <div>
      <MyDialog
        onClick=(self.reduce((_event) => Click))
        onSubmit=(self.reduce((_event) => Toggle))
      />
      {ReasonReact.stringToElement(message)}
    </div>
  }
};

一些信息：

  • There's a user-defined type called action, named so by convention. 这是一个 variant 类型，它描述了所有可能的 state 的变化的方式。 在状态管理的概念里面，你也可以把它想象成一种 "token"
  • 一个用户定义的 state 类型，以及一个 initialState ，没什么特别的。
  • 组件当前的 state 可以通过 self.state 拿到。 不管self 是被当作参数传递或者用它本身的某些方法 。
  • 一个"reducer"！ 这是一种 pattern-matches 模式，会覆盖所有可能的 actions 并且对于每个 action 反应如何去更新 state。 *在状态管理的概念里面，它就是一种 "state transition" *
  • 现在，在 render 函数里，我们将用 self.reduce 取代 self.handle。 reduce 接受一个 callback 作为参数，给这个 callback 传递一个事件（或者像 MyDialog 里面 onSubmit/onClick/onFoo 传递给你的任何参数） 然后需要你返回一个 action

所以，当 MyDialog 的click事件被触发的时候，我们会触发一个 "Click" action 给组件的 reducer，然后 reducer 会处理这个 "Click" action 并返回一个新的 state。 ReasonReact 会负责用新的 state 去更新组件。

注意： 和 self.handle 一样， 有时候你需要把 reduce 传递给一些其它的函数或者方法。 请传递整个self并标记它的类型。 这可以避免self里复杂的类型标记行为。 参考Record Field reduce/handle Not Found.

用 Reducer 更新状态

注意到 reducer 的返回值了么？ 请注意 ReasonReact.Update。 和直接返回一个新的 state 不同，ReasonReact会让你返回一个被 "Update" 这个 Variant 包裹的 state。 下面是（这个 Variant）可能的值：

  • ReasonReact.NoUpdate: 不要更新state。
  • ReasonReact.Update state: 更新state。
  • ReasonReact.SideEffects((self) => unit): no state update, but trigger a side-effect, e.g. ReasonReact.SideEffects((_self) => Js.log("hello!")).
  • ReasonReact.UpdateWithSideEffects(state, (self) => unit): update the state, then trigger a side-effect.

如果你对ReasonReact很熟悉，这里你还可以使用 SilentUpdate 和 SilentUpdateWithSideEffects。 请参阅 reasonReact.rei，来查看如何使用它们。 但是不要试图使用它们去更新一些像 ref/timer/subscription 之类的实例变量（Instance variable 值可以改变的变量）。

重要提示

请阅读以下所有内容，如果你想更好的使用 reducer 以及避免未来在 ReactJS Fiber 中遇到race condition的问题：

  • The action type's variants can carry a payload: onClick=(self.reduce((data) => Click(data.foo))).
  • 不要直接把一个事件本身当成参数传递给 action。ReactJS中事件是在一个事件池中的，当你试图在 reducer 中得到它时，它已经被回收了。
  • reducer must be pure (not to be confused with self.reduce, which can be impure)! Aka don't do side-effects in them directly. 在将来 React Fiber 可以并发渲染时你会感谢我们这么做。 用 SideEffects 或者 UpdateWithSideEffects 去安排一个 side-effect， 这个 side-effect 会在 state 处理完之后，下一次渲染之前进行。
  • 如果你要用到类似于 ReactEventRe.BlablaEvent.preventDefault event（注：JS里面的event.preventDefault），请在 self.reduce 方法返回 action 之前使用，再强调一遍， reducer 必须要是纯函数。
  • Feel free to trigger another action in SideEffects and UpdateWithSideEffects, e.g. UpdateWithSideEffects(newState, (self) => self.reduce(() => Click), ()).
  • If your state only holds instance variables, it also means (by the convention in the instance variables section) that your component only contains self.handle, no self.reduce. You still needs to specify a reducer like so: reducer:((), _state) => ReasonReact.NoUpdate. Otherwise you'll get a variable cannot be generalized type error.

提示

尽可能的往 reducer 里面多塞东西， Keep your actual callback handlers (the self.reduce((foo) => Bar)) part) dumb and small. 这会让你的状态更新以及副作用处理（因为它们只能在 ReasonReact.SideEffects 和ReasonReact.UpdateWithSideEffects中进行）更清楚明白， 也更适应 ReactJS filber 的异步模式。

异步状态更新

在 ReactJS，你可以在任何方法里使用 setState，像这样：

setInterval(() => this.setState(...), 1000);

在ReasonReact里，只要有新的状态，肯定是通过 reducer 返回的，所以上面的写法行不通。而且，在 setInterval 里面返回一个新的状态也没有什么意义！

所以，你会这么做：

type action =
  | Tick;

type state = {
  count: int,
  timerId: ref(option(Js.Global.intervalId))
};

let component = ReasonReact.reducerComponent("Counter");

let make = (_children) => {
  ...component,
  initialState: () => {count: 0, timerId: ref(None)},
  reducer: (action, state) =>
    switch action {
    | Tick => ReasonReact.Update({...state, count: state.count + 1})
    },
  didMount: (self) => {
    self.state.timerId := Some(Js.Global.setInterval(self.reduce((_) => Tick), 1000));
    ReasonReact.NoUpdate
  },
  render: ({state}) => <div> {ReasonReact.stringToElement(string_of_int(state.count))} </div>
};

这就是说，先创建一个 reducer 正常的处理你的逻辑，然后让 setInterval （或者别的异步工作流程）异步地去调用你的 self.reduce。
