与现有的 ReactJS 组件交互

在 ReasonReact 中使用 ReactJS 组件

很简单！既然 Reason 组件需要暴露一个 make 函数，写一个就行了：

external myJSReactClass : ReasonReact.reactClass = "myJSReactClass" [@@bs.module];

let make name::(name: string) age::(age: option int)=? children =>
  ReasonReact.wrapJsForReason
    reactClass::myJSReactClass
    props::{"name": name, "age": Js.Nullable.from_opt age}
    children;

ReasonReact.wrapJsForReason 是我们为此设计的一个辅助函数。 它的参数有：你想要使用的 reactClass， 你需要传下去的 props (类型是 Js.t {. foo: bar}，数据类型需要从 Reason 的类型转换成对应的 JS 数据类型），以及如果你需要的话传到 JS 那边的 children。

我们建议 定义好 make 函数参数的数据类型，因为他们会作为 props 传入 JS 中，那边不会规定参数的类型。

在 ReactJS 中使用 ReasonReact 组件

非常简单，我们暴露了一个方法 ReasonReact.wrapReasonForJs：

let component = ...;
let make ...;

let default =
  ReasonReact.wrapReasonForJs
    ::component
    (fun jsProps => make name::jsProps##name age::?(Js.Nullable.to_opt jsProps##age) [||]);

这个函数接受两个参数，第一个是 labeled argument，就是你刚才构造的 reason component ，第二个参数是一个函数，这个函数接受 JS props，然后需要你去调用 make 函数，同时将 JS props 的类型转换成 reason 数据类型。 你只要将所有东西 （这个函数）命名为 default， 然后 JS 那边就可以直接拿到：

var MyReasonComponent = require('myReasonComponent');
// make sure you're passing the correct data types!
<MyReasonComponent name="John" />

注意： default 语法直到 BuckleScript >=1.8.2 版本才被支持，如果你发现这样写不起作用，试着用一个非 default 的导出方式：

let jsComponent = ReasonReact.wrapReasonForJs ...

然后在 JS 的代码中：

var MyReasonComponent = require('myReasonComponent').jsComponent
