Style

现在，CSS-in-JS 似乎成为了众矢之的，我们也将尽快推出官方建议的解决办法。在这同时，如果你要写 inline style，我们提供了 ReactDOMRe.Style.make API：

<div style=(
  ReactDOMRe.Style.make
    color::"#444444"
    fontSize::"68px"
    ()
)/>

这是一个 labeled function (typed!)，会映射成我们熟悉的样式对象 {color: '#444444', fontSize: '68px'}。 注意 这里的make 函数返回的是一个封闭的 ReactDOMRe.style 类型，你并不能直接读取， 所以我们还暴露了一个 ReactDOMRe.Style.combine ，可以接受两个 style 类型然后合并它们。

Escape Hatch: unsafeAddProp #

上面的 Style.make API 会检查每个 style 字段的类型！ 但是，我们很可能忽略了一些更复杂的情况。 在有的情况下，类型检查系统会告诉你你想要添加的字段不存在。 为了解决这个问题，我们还暴露了一个 ReactDOMRe.Style.unsafeAddProp 让你可以跳过类型检查危险的添加一个字段：

let myStyle = ReactDOMRe.Style.make color::"#444444" fontSize::"68px" ();
let newStyle = ReactDOMRe.Style.unsafeAddProp myStyle "width" "10px";
