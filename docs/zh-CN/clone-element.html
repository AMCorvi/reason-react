cloneElement

定义： let cloneElement: reactElement => props::Js.t {..}? => 'anyTypeHere => reactElement；

ReactJS 的 cloneElement 相同。 但是，在 ReasonReact 中增加新的 props 是没有意义的，你应该做的是柯里化或者通过别的途径。 因此，ReasonReact.cloneElement 只用于处理一些特殊的情况。 比如，data-* 和 aria-* 标签并不能作为函数的 labeled argument， 下面这样写不能被正确编译：

<div data-payload=1 aria-label="click me" className="foo" />

你可以使用 cloneElement 来绕过这个情况：

let myElement =
  ReasonReact.cloneElement
    <div className="foo" />
    props::{"data-payload": 1, "aria-label": "click me"}
    [||];

这种方式通过一个聪明的，语义化的 Reason 语法，给 div 增加另外的 data-payload 和 aria-label 属性 （注意，没有标注类型）。

对于非 DOM 组件, 你需要使用有效的 prop 命名。

你也可以使用 cloneElement 来模拟 prop spread , 但这在 ReasonReact 中是不鼓励的。
