处理回调

这个章节描述了像 ReactJS 中<div onClick={this.handleClick} /> 的代码在ReasonReact中是如何处理的。

没有State更新的回调 #

两个场景

不需要读取self中的数据 #

记住：self就是ReasonReact中的this。它是一个包含了 state, retainedProps，等字段的一个Record。

如果你只是要传递一个回调方法到某个child上面，你需要做的和在ReactJS中是一样的：

let component = ...;

let make ::name ::onClick _children => {
  ...component,
  render: fun self => <button onClick=onClick />
};

不用觉得奇怪。 Reason的JSX语法包含了 punning syntax,，所以button 会转换成 <button onClick />。

同样的，如果需要预处理某个值再传递给调用组件的地方：

let component = ...;

let make ::name ::onClick _children => {
  let click event => onClick name; /* pass the name string up to the owner */
  {
    ...component,
    render: fun self => <button onClick=click />
  }
};

读取self中的数据 #

要拿到state, retainedProps或者self中别的信息，你需要用self.handle去调用你的回调函数：

let component = ...;
let make ::name ::onClick _children =>
  let click event self => {
    onClick event;
    Js.log self.state;
  };
  {
    ...component,
    initialState: ...,
    render: fun self => <button onClick=(self.handle click) />
  }
};

注意 你的 click 方法现在接受另外一个 self 参数。 一般来收， self.handle 需要具有如下特性的回调方法：

  • 接受一个直接传递进来的参数（payload），比如：handleClick.
  • 参数最后会多一个self.
  • 没有返回值（返回类型是()或者unit）

注意 2 : 有时候你可能会想将 handle 传递给一些别的函数， 这时请传递整个self并标记它的类型。 这可以避免self里复杂的类型标记行为。 See Record Field reduce/handle Not Found.

说明 #

事实上， self.handle 只是一个普通的函数，它接受两个参数，第一个是回调函数本身，第二个参数是要传给回调函数的那个数据（payload）。

明白了吗？ 由于Reason语言天然的柯里化属性，我们通常只要你传递第一个参数。 然后这会返回一个新的函数，新的函数会接受第二个参数，然后执行函数体。 第二个参数（回调里的self）是由函数的caller传进来的，就是你正在rendering的组件本身！

接受多个参数的回调 #

有时候，你在调用的组件是写在JS代码里的（可以参考 ReasonReact<->ReactJS interop），它可能需要在回调的方法里接收不止一个参数。 在 ReactJS中，它会看起来像：

handleSubmit: function(username, password, event) {
  this.setState(...)
}
...
<MyForm onUserClickedSubmit={this.handleSubmit} />

你不能直接在ReasonReact里面写handleSubmit这样的方法，因为 handle 包裹的函数只能接收一个参数。下面是处理这种情况的方法:

let handleSubmitEscapeHatch username password event => self.handle (fun tupleOfThreeItems => doSomething tupleOfThreeItems) (username, password, event);
...
<MyForm onUserClickedSubmit=(handleSubmitEscapeHatch) />

总的来说，我们写的回调方法需要：

  • 接受从JS component方法里传来的若干参数
  • 把接受的参数包装成一个 tuple 然后调用 self.handle
  • 构造另一个普通函数（接受一个 tuple）传递给 handle
  • 调用一次 self.handle （第一个参数是接受一个 tuple 的函数，第二个参数是我们手工pass进去的 tuple）
